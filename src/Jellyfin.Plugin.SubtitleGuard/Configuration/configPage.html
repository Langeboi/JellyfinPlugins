Subtitle Guard

Standardizes subtitle size in the web/mobile player and watches every playback to make sure selected subtitles actually render - re-applying them automatically when they silently fail.

One consistent, viewport-scaled size on every device, instead of whatever each player defaults to.
100 = the standard size. Raise or lower to taste (50-200).
While something plays with subtitles selected, verifies they are actually being shown - and re-applies the subtitle track automatically if not. Only applies to text subtitles (burned-in image subtitles can't fail this way).

Sync workers

Machines that do the actual subtitle work. Enroll one by running this on any Debian/Ubuntu machine that has the media mounted:

curl -sL https://raw.githubusercontent.com/Langeboi/JellyfinPlugins/main/worker/subtitle-worker/install.sh | sudo bash

The installer prints the worker's URL and an enrollment code - paste them below. Jobs are split between whichever enrolled workers are online (machines that are off are simply skipped).

How Jellyfin sees the media path. Leave both fields empty if the worker machine mounts media at the same paths.
The same location as seen from the worker machine.
